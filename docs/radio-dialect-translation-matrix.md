# Radio Dialect Translation Matrix

Phase 0 translation target for the radio dialect scheduler refactor.

## Translation Rules

The radio modules translate between public normal HILink and compact RF
dialect. FC and GCS code never construct RF frames.

Bridge helpers in the protocol update are the intended pure translation core:

| Helper | Direction |
| --- | --- |
| `telemetry_to_lora_flight` | normal `TelemetrySnapshot` -> RF `LoRaFlightSnapshot` |
| `gps_to_lora_gps` | normal `Gps` -> RF `LoRaGpsSnapshot` |
| `normal_command_to_lora` | supported normal command identity -> RF `LoRaCommand` plus correlation metadata |
| `lora_command_to_normal` | RF `LoRaCommand` -> supported normal command identity |
| `ack_to_lora_command_ack` | normal `Ack`/`Nack` -> RF `LoRaCommandAck` using correlation metadata |
| `lora_command_ack_to_normal` | RF `LoRaCommandAck` -> normal `Ack`/`Nack` using correlation metadata |

Unsupported conversions return `UnsupportedTranslation`. ACK conversion returns
`CorrelationMismatch` when a normal ACK/NACK does not match the radio bridge's
stored normal/RF command correlation.

## Normal To RF

| Normal HILink input | RF output | Priority | Reliability/freshness | Notes |
| --- | --- | ---: | --- | --- |
| `Disarm` | `LoRaCommand { command_id: DISARM }` | P0 | reliable, immediate | Safety command. Should preempt telemetry. |
| `MotorStop` | `LoRaCommand { command_id: MOTOR_STOP }` | P0 | reliable, immediate | Valid even when bench mode is disabled on the FC side. |
| normal abort source, if represented by policy | `LoRaCommand { command_id: ABORT }` | P0 | reliable, immediate | There is no separate normal `Abort` `MsgType`; bridge policy must define the normal trigger before use. |
| `Arm` | `LoRaCommand { command_id: ARM }` | P1 | reliable command | Use normal seq plus RF `command_seq` correlation. |
| `Ping` | `LoRaCommand { command_id: PING }` or local radio health probe | P1 | reliable command | For end-to-end health, translate to RF. For local radio-module health, answer locally. |
| `Rtl` | policy-defined `LoRaCommand` or unsupported | P1 | reliable command | V1 RF command IDs do not include explicit RTL. Add an RTL RF ID or document mapping to `SET_MODE`/recovery before enabling. |
| mode/rate/profile normal command, if added | `LoRaCommand { SET_MODE, SET_TELEMETRY_RATE, SET_RADIO_PROFILE }`, `LoRaSetProfile` | P1 | reliable command | `LoRaSetProfile` is compact and strongly typed for common radio profile changes. |
| snapshot request normal command, if added | `LoRaRequestSnapshot` or `LoRaCommand { REQUEST_* }` | P1 | reliable command | Request flags select flight/GPS/fault snapshots. |
| supported waypoint command | unsupported in V1 unless mapped by policy | P1 | reliable command | RF V1 has no waypoint-specific command IDs; do not silently compress mission semantics. |
| `Ack` / `Nack` for a correlated normal command | `LoRaCommandAck` | P1 | reliable acknowledgement | Requires bridge correlation metadata. |
| `TelemetrySnapshot` | `LoRaFlightSnapshot` | P3 | best-effort latest-only | Lossy compression of state, mode, flags, altitude, vertical velocity, accel magnitude, battery, actuator/pyro flags, fault summary. |
| `Gps` | `LoRaGpsSnapshot` | P4 | best-effort latest-only slow | Separate lower-rate GPS stream. Invalid GPS uses RF invalid sentinels and clears `GPS_VALID`. |
| normal fault report, when present | `LoRaFaults` | P2 | on-change or latched | Current supplied normal `MsgType` table has no explicit fault payload; bridge can derive summary from system/fault service once exposed. |
| normal event report, when present | `LoRaEvent` | P2 | on-change queued | Current supplied normal `MsgType` table has no explicit event payload; add/identify normal source before implementation. |
| link health from local radio | `LoRaLinkStatus` | P4 | best-effort slow | Generated by radio bridge firmware, not by FC/GCS. |

## RF To Normal

| RF input | Normal HILink output | Priority | Reliability/freshness | Notes |
| --- | --- | ---: | --- | --- |
| `LoRaCommand { DISARM }` | `Disarm` | P0 | reliable, immediate | Vehicle radio writes normal HILink command to FC. |
| `LoRaCommand { MOTOR_STOP }` | `MotorStop` | P0 | reliable, immediate | Vehicle radio writes normal HILink command to FC. |
| `LoRaCommand { ABORT }` | policy-defined normal command | P0 | reliable, immediate | Requires a normal abort representation; do not invent one inside FC/GCS code. |
| `LoRaCommand { ARM }` | `Arm` | P1 | reliable command | Preserve normal/RF correlation for subsequent ACK/NACK. |
| `LoRaCommand { PING }` | `Ping` or local radio response | P1 | reliable command | End-to-end health uses normal `Ping`; local radio health may terminate at radio module. |
| `LoRaCommand { SET_MODE, ENTER_RECOVERY_BEACON }` | policy-defined normal command or local radio action | P1 | reliable command | Some RF commands are radio-management only and should not be forwarded to FC. |
| `LoRaSetProfile` | local radio profile action | P1 | reliable command | Radio-internal management. Do not expose to FC as RF dialect. |
| `LoRaRequestSnapshot` | local scheduler request | P1 | reliable command | Triggers local cached snapshot generation where possible. |
| `LoRaCommandAck` | normal `Ack` or `Nack` | P1 | reliable acknowledgement | Uses stored correlation to restore normal `acked_seq`/`acked_msg_type` or `rejected_seq`/`rejected_msg_type`. |
| `LoRaFlightSnapshot` | `TelemetrySnapshot` and/or `Heartbeat`/`SystemState` | P3 | best-effort latest-only | GS radio expands compact telemetry for GCS. Fields not present in RF must be marked invalid, zeroed by policy, or omitted through a narrower normal message. |
| `LoRaGpsSnapshot` | `Gps` | P4 | best-effort latest-only slow | Rehydrate scaled fields; invalid sentinels become invalid/zero GPS fields according to normal HILink validity policy. |
| `LoRaFaults` | normal fault/status report | P2 | on-change or latched | Use a normal fault report once available; otherwise translate to a documented status payload, not an RF dialect leak. |
| `LoRaEvent` | normal event/status report | P2 | on-change queued | Use normal event reporting once available. |
| `LoRaLinkStatus` | normal radio/link status report | P4 | best-effort slow | GS radio owns downlink metadata display to GCS. Vehicle radio owns uplink/downlink measurements from its perspective. |

## No Default RF Translation

| Normal message | Default decision |
| --- | --- |
| `HilSensorFrame`, `HilResponseFrame`, `HilReady` | Wired/local HIL only. RF may reject, throttle, summarize, or enter a special debug mode by explicit policy. |
| `Imu`, `Mag`, `Baro`, `EstimatorState`, `MotorState` | Do not forward raw high-rate detail by default. Coalesce into snapshots or requested debug only. |
| `BenchEnable`, `BenchDisable`, `MotorTest`, `MotorSweep`, `DshotCommand` | Keep local/wired until an RF bench safety policy is explicitly approved. |
| `ControlWaypoint`, `CvWaypoint`, `TofWaypoint`, `MissionWaypoint` | Unsupported in RF V1 unless a semantic compact command mapping is added. |
| `Battery`, `RadioStatus` | Reserved/incomplete normal IDs in the supplied map. Use only after payload contract is finalized. |

## Correlation Metadata

For every normal command translated to RF, the bridge stores:

| Field | Purpose |
| --- | --- |
| normal `Header::seq` | Rebuild normal ACK/NACK for GCS/FC. |
| normal `MsgType` | Rebuild normal ACK/NACK and reject mismatched responses. |
| RF `command_id` | Match RF ACK and duplicate cache entries. |
| RF `command_seq` | Per-GS radio monotonic command sequence. |
| enqueue/send time | Enforce expiry and profile retry windows. |
| original result, when complete | Return duplicate accepted/rejected status without reapplying stale commands. |

V1 duplicate cache depth is 16 command results or 30 seconds, whichever expires
first.

## Compact Payload Budget

| RF payload | Payload bytes | RF frame bytes including type and CRC |
| --- | ---: | ---: |
| `LoRaRequestSnapshotPayload` | 4 | 7 |
| `LoRaSetProfilePayload` | 8 | 11 |
| `LoRaCommandAckPayload` | 12 | 15 |
| `LoRaFaultsPayload` | 14 | 17 |
| `LoRaEventPayload` | 15 | 18 |
| `LoRaCommandPayload` | 16 | 19 |
| `LoRaLinkStatusPayload` | 18 | 21 |
| `LoRaFlightSnapshotPayload` | 22 | 25 |
| `LoRaGpsSnapshotPayload` | 22 | 25 |

Conclusion: V1 compact RF payloads are small enough for the intended LoRa
scheduler budget. Normal COBS/HILink frames, especially HIL and component
sensor frames, are not the RF budget unit for the refactored bridge.
